# 画面設計

## 画面遷移図

```
[トップ画面]
     │
     ├─ 「部屋を作る」→ [部屋作成画面] → [待機画面（ホスト）]
     │                                         │
     └─ 「URLから参加」→ [待機画面（ゲスト）] ─┘
                                               │
                                    ホストが「投票開始」を押す
                                               │
                                    [ジャンルスワイプ画面]
                                               │
                                    全員スワイプ完了
                                               │
                                    [店舗スワイプ投票画面]
                                               │
                                    全員スワイプ完了
                                               │
                                    [結果発表画面]
```

---

## 各画面詳細

### 1. トップ画面 `/`

**目的:** アプリへの入口。部屋の作成 or 参加。

**UI要素:**
- アプリロゴ・キャッチコピー
- 「部屋を作る」ボタン（主アクション）
- 「使い方を見る」リンク → チュートリアルモーダル表示
- URLパラメータに `room_id` がある場合は自動的に参加フローへ

**チュートリアルモーダル（初回のみ自動表示）:**
1. ジャンルをスワイプで選ぶ
2. 近くの店をスワイプで投票
3. 一番人気の店が決定！
- 「わかった！」で閉じる・localStorageで初回判定

---

### 2. 部屋作成画面 `/room/new`

**目的:** 幹事が検索条件を設定してセッションを作る。

**UI要素:**
- 現在地取得ボタン（Geolocation API）
- 場所名テキスト表示（取得後）
- 予算セレクト（ホットペッパー予算コード）
- 最大参加人数スライダー（2〜20人、デフォルト10）
- 「部屋を作る」ボタン

**バリデーション:**
- 位置情報未取得の場合は作成不可・エラーメッセージ表示

---

### 3. 待機画面 `/room/[id]`（status: waiting）

**目的:** 参加者が揃うまで待つ。LINEでURLをシェアする。

**UI要素（ホスト・ゲスト共通）:**
- 参加者数カウンター「5人が参加中」
- LINEシェアボタン（URLスキーム）
- QRコード表示（URLのQR）
- コピーボタン（URL）
- セッション有効期限カウントダウン
- 参加者アイコン一覧（匿名・番号表示）

**UI要素（ホストのみ追加）:**
- 「投票を開始する」ボタン（2人以上で有効化）
- 部屋設定サマリー（場所・予算）

---

### 4. ジャンルスワイプ画面 `/room/[id]/genre`（status: genre_voting）

**目的:** 各自が行きたいジャンルをスワイプ選択する。

**UI要素:**
- スワイプカード（1枚ずつ表示）
  - ジャンル名（大文字）
  - ジャンルアイコン・イメージカラー
- 操作説明バー（右=あり / 左=なし / 上=許容範囲内）
- 残り枚数インジケーター「3/8」
- 投票進捗「5人中2人が完了」（Realtimeで更新）
- 全員完了後 → 自動的に次画面へ遷移

**スワイプ判定:**
- 右スワイプ / ❤️ボタン → `yes`
- 上スワイプ / 🤔ボタン → `maybe`
- 左スワイプ / ✕ボタン → `no`

**ジャンル一覧（ホットペッパー準拠）:**
居酒屋 / ダイニングバー / 焼肉・ホルモン / 鍋 / 寿司 / 焼き鳥・串揚げ / 中華 / イタリアン・フレンチ / アジア・エスニック / ラーメン

---

### 5. 店舗スワイプ投票画面 `/room/[id]/vote`（status: shop_voting）

**目的:** 候補店舗を1枚ずつスワイプして投票する。

**UI要素:**
- スワイプカード（ShopCard）
  - 店舗写真（フルカード背景）
  - 店名・ジャンル
  - 予算帯・徒歩分数
  - キャッチコピー（口コミ代替）
  - 営業時間・深夜営業バッジ
  - 予約可否バッジ
  - 席数
- 操作説明バー（右=あり / 上=許容 / 左=なし）
- 残り枚数インジケーター
- 投票進捗バー「5人中3人が投票完了」（Realtimeで更新）

**ローディング状態:**
- ホットペッパーAPI取得中はスケルトン表示

---

### 6. 結果発表画面 `/room/[id]/result`（status: finished）

**目的:** 最多票の店を発表し、予約へ誘導する。

**スコア計算:**
- `yes` = 2点、`maybe` = 1点、`no` = 0点
- 同点の場合はランダム選出（「みんなで選びました！」と表示）

**UI要素:**
- 🎉 決定店舗カード（大）
  - 店名・写真・ジャンル
  - スコアバー（yes/maybe/noの内訳）
  - 徒歩分数・予算帯
  - 「ホットペッパーで予約する」ボタン
  - 「電話する」ボタン
- 次点ランキング（2位・3位を小さく表示）
- 「もう一度投票する」ボタン（ホストのみ）→ ステータスをリセット
- 「LINEでシェア」ボタン → 決定店舗名・URLをシェア

---

## スマホUI共通ルール

- 最小タップ領域: 44×44px
- フォントサイズ最小: 16px（酔っていても読める）
- スワイプカードサイズ: 画面幅90%・高さ70vh
- ボトムナビゲーションなし（ステップが明確なため）
- カラーテーマ: 温かみのあるオレンジ系（食欲を刺激）
