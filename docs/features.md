# 機能仕様

## 1. セッション管理

### 部屋の作成
- 幹事はGeolocation APIで現在地を取得して部屋を作成する
- 作成時に予算・最大参加人数を設定
- 作成完了後、一意のUUIDがURLに付与される（例: `/room/abc123`）
- セッションの有効期限は作成から **2時間**
- 有効期限切れのセッションにアクセスした場合はトップ画面へリダイレクト

### 部屋への参加
- URLをLINEでシェアすると、リンクを開いた人が自動的にゲストとして参加
- 参加者はSupabaseの **匿名認証** でUIDを発行（ログイン不要）
- UIDはlocalStorageに保持し、同じ端末・ブラウザではセッションが継続
- 最大参加人数を超えた場合は参加不可メッセージを表示

### ホスト権限
- 幹事（作成者）のみが以下の操作を実行可能：
  - 投票の開始
  - 投票のリセット（結果画面から再投票）
- ホスト判定: `room.host_user_id === 現在のuser_id`

## 2. リアルタイム同期

### 投票進捗
- Supabase Realtimeで `shop_votes` テーブルのINSERTを監視
- 「5人中3人が投票完了」のような進捗を全参加者にリアルタイム表示
- 全参加者の投票完了を検知した時点で自動的に次のステップへ遷移

### ステータス遷移（ホストのみが実行）
```
waiting → genre_voting（ホストが「投票開始」を押す）
genre_voting → shop_voting（全員のジャンル投票完了を検知）
shop_voting → finished（全員の店舗投票完了を検知）
```
- ステータス変更はホスト端末が `rooms` テーブルを UPDATE
- 他参加者はRealtimeで変更を受け取り、自動遷移

## 3. 匿名投票

### 投票方法
- スワイプ操作: 右（あり）/ 上（許容範囲内）/ 左（なし）
- ボタン操作: ❤️ / 🤔 / ✕（スワイプが難しい場合の代替）

### 集計ロジック
```
スコア = yes数 × 2 + maybe数 × 1 + no数 × 0
```
- 最高スコアの店舗を結果として表示
- 同点の場合: サーバーサイドでランダム選出（`Math.random()`）
- 結果画面でyes/maybe/noの内訳グラフを表示（個人の投票は非公開）

### 匿名性の保証
- 個人の投票内容は他参加者に非公開
- 集計後の内訳（yes:3 / maybe:2 / no:0）のみ表示
- DBのRLSで他ユーザーの投票レコードは参照不可

---

## 4. 店舗情報取得（ホットペッパーAPI）

### 検索フロー
1. ジャンル投票の結果を集計（`yes`が多いジャンル上位3つを取得）
2. 部屋の `location_lat/lng` と `budget` でホットペッパーAPIを呼び出し
3. 取得した店舗（最大20件）を `shops` テーブルに保存
4. 以降はDBから読み取り（APIコールは1セッション1回のみ）

### 表示情報
| 項目 | ソース |
|------|--------|
| 店名 | ホットペッパー `name` |
| ジャンル | ホットペッパー `genre.name` |
| 予算 | ホットペッパー `budget.average` |
| 写真 | ホットペッパー `photo.pc.l` |
| キャッチコピー | ホットペッパー `catch` |
| アクセス | ホットペッパー `access` |
| 営業時間 | ホットペッパー `open` |
| 深夜営業 | ホットペッパー `open` から23時以降を判定 |
| 席数 | ホットペッパー `capacity` |
| 予約可否 | ホットペッパー `reservation_url` の有無 |
| 予約URL | ホットペッパー `urls.pc` |

## 5. LINE・シェア機能

### LINEシェア（URLスキーム）
```
https://line.me/R/msg/text/?【店名】に決まりました！%0A詳細はこちら%0A{URL}
```
- 待機画面: 「参加してね！」テキスト + 部屋URL
- 結果画面: 「【店名】に決まりました！」テキスト + 部屋URL

### QRコードシェア
- 待機画面にQRコード表示（`qrcode` ライブラリで生成）
- 「QRを保存」ボタン（canvas → PNG download）

### URLコピー
- クリップボードコピー（`navigator.clipboard.writeText`）
- コピー後2秒間「コピーしました！」トースト表示

## 6. セッション期限・Supabase停止対策

### 有効期限管理
- `expires_at` = 作成時刻 + 2時間
- APIアクセス時・画面表示時に期限チェック
- 期限切れ: トップ画面にリダイレクト + 「このセッションは終了しました」メッセージ

### Supabase自動停止対策
- Vercelの無料Cron（`vercel.json`）で1日1回ダミークエリを実行
- Cron設定: `"crons": [{ "path": "/api/ping", "schedule": "0 9 * * *" }]`
- `/api/ping` はSupabaseにSELECT 1を発行するだけのエンドポイント

## 7. チュートリアル

### 表示条件
- `localStorage` の `tutorial_shown` キーが存在しない場合に初回表示
- トップ画面でモーダル表示
- 「使い方を見る」リンクからいつでも再表示可能

### 内容（3ステップのスライド）
1. 「ジャンルをスワイプ」 → ジャンルカードのアニメーション例
2. 「近くの店に投票」 → 店舗カードのスワイプ例
3. 「みんなの票で決定！」 → 結果画面のプレビュー

## 8. エラーハンドリング

| エラー | 対処 |
|--------|------|
| 位置情報の取得失敗 | 手動で場所名を入力するフォールバック（住所検索は未実装・手入力） |
| ホットペッパーAPI失敗 | リトライボタン表示、最大3回自動リトライ |
| Supabase接続失敗 | トースト通知 + 10秒後自動リトライ |
| セッション期限切れ | トップ画面リダイレクト |
| 最大参加人数超過 | 参加不可メッセージ |
| 同一ユーザーの二重投票 | DBのユニーク制約でブロック（UIでも投票済みはスキップ） |
