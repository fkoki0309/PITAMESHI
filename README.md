# PITAMESHI 🍻

2次会の飲食店をみんなでリアルタイム匿名投票で決めるスマホWebアプリ。

## 使い方

### 1. 幹事が部屋を作る
1. トップ画面で「部屋を作る」をタップ
2. 現在地を取得し、予算・最大参加人数を設定
3. 待機画面のURLをLINEでシェア or QRコードを見せる

### 2. 参加者が入室する
- LINEのURLを開くだけで自動参加（ログイン不要）
- 全員が揃ったらホストが「投票を開始する」をタップ

### 3. ジャンルをスワイプで投票
| 操作 | 意味 |
|------|------|
| 右スワイプ or ❤️ | 行きたい |
| 上スワイプ or 🤔 | まあいい |
| 左スワイプ or ✕ | いらない |

全員がスワイプを完了すると、得票数の多いジャンル上位3つが自動集計される。

### 4. 近くの店舗をスワイプで投票
現在地・予算・ジャンルをもとにホットペッパーから取得した店舗をスワイプで投票する。

### 5. 結果発表
全員の投票が揃うと自動で結果画面へ。最高得点の店舗が決定店舗として表示される。
- ホットペッパーで予約
- LINEで結果をシェア
- ホストは「もう一度投票する」で再投票可能

---

## 集計ロジック

```
スコア = YES票 × 2 + MAYBE票 × 1
```

同点の場合はルームIDとショップIDのハッシュで順位を決定（全参加者で一致）。

## セットアップ

### 必要なもの
- Node.js 18以上
- Supabaseプロジェクト
- ホットペッパーAPIキー

### 環境変数

`.env` を作成して以下を設定：

```
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=
SUPABASE_SERVICE_ROLE_KEY=
HOTPEPPER_API_KEY=
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

### DBセットアップ

Supabaseの SQL Editor で `supabase/schema.sql` を実行する。

### 開発サーバー起動

```bash
npm install
npm run dev
```

## 仕様

### セッション
- 有効期限：作成から20分
- 最大参加人数：2〜20人（作成時に設定）
- 期限切れのセッションにアクセスするとトップへリダイレクト

### 匿名性
- 参加者はログイン不要（Supabase匿名認証）
- 個人の投票内容は非公開。結果画面にはYES/MAYBE/NOの内訳のみ表示

### 離脱検知
- ハートビート（10秒間隔）でアクティブ参加者を管理
- 25秒間応答がない参加者は離脱扱いになり参加者数から除外
- 離脱した参加者の分は投票完了判定から除外されるため、残った参加者で投票を進められる
